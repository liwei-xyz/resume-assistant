# resume-structure-guide - 通用简历结构指导技能

## 触发条件

当用户提到以下关键词时激活此技能：
- "简历结构"、"简历模板"、"简历怎么写"
- "简历应该包含"、"简历格式"、"简历框架"
- "优化简历结构"、"简历怎么写更好"
- 用户发送简历后问"怎么改进结构"

**注意**：如果用户明确要求"优化我的简历"并附上了简历内容，优先使用 `resume-optimizer` 技能生成精美 HTML 简历。本技能侧重于**结构指导和原则说明**。

---

## 核心任务

帮助用户理解**通用简历结构**，适用于不同经验水平（应届/初级/中级/高级）和不同岗位（程序/策划/美术/测试/运营）。

---

## 执行流程

### 1. 先问清楚（如果用户没说）

- 目标岗位是什么？（程序/策划/美术/测试/运营）
- 经验水平？（应届/初级 1-3 年/中级 3-5 年/高级 5 年+）
- 是否有现有简历需要诊断？

### 2. 输出通用结构框架

按照以下 6 部分讲解：

#### 📋 核心原则（先说）

不管什么岗位，简历就回答 3 个问题：
1. 你是谁？（基本信息 + 定位）
2. 你能做什么？（技能 + 经验）
3. 你做成过什么？（成果 + 数据）

#### 📑 标准结构（6 部分）

**1. 基本信息**（必填，简洁）
```
姓名 | 求职意向 | 城市
电话 | 邮箱 | 微信/GitHub/作品集链接
```
- 求职意向要具体
- 邮箱用专业的
- 有作品集一定要放链接
- 不需要：性别、年龄、照片（除非明确要求）、详细地址

**2. 个人概述/求职总结**（3-5 句话，黄金位置）

公式：`[年限] + [核心技能] + [代表项目/成就] + [差异化优势]`

不同经验水平写法：
| 经验 | 重点 | 示例 |
|------|------|------|
| 应届/初级 | 学习能力 + 项目热情 | "计算机专业本科，熟悉 Unity/C#，独立完成 3 个游戏 Demo，GitHub 200+ stars" |
| 中级 (3-5 年) | 技术深度 + 项目成果 | "3 年客户端开发，主导 XX 项目渲染模块优化，帧率提升 40%" |
| 高级 (5 年+) | 架构能力 + 团队贡献 | "8 年经验，搭建多项目通用框架，培养 5+ 初级工程师" |

**3. 工作经历**（倒序，最近的放前面）

格式：
```
公司名称 | 职位 | 时间范围
- 核心职责（1-2 条）
- 关键成果（2-4 条，用数据说话）
```

要点：
- ✅ 用动词开头（负责/主导/搭建/优化/实现）
- ✅ 成果尽量量化（数字、百分比、时间）
- ✅ 写与求职岗位相关的经历

没工作经验：写实习/课程设计/毕业设计/个人项目/GameJam

**4. 项目经历**（核心部分，详细写）

格式：
```
项目名称 | 你的角色 | 时间范围 | 项目规模（可选）
项目简介：1 句话说明是什么游戏/产品
技术栈：列出核心技术/工具
关键贡献：
- 你具体做了什么
- 遇到什么难点
- 怎么解决的
- 结果如何（数据/效果）
```

要点：
- ✅ 项目简介让外行也能看懂
- ✅ 技术栈写具体的
- ✅ 用 STAR 法则：情境→任务→行动→结果

**5. 技能清单**（分类写，别堆砌）

格式：
```
编程语言：Go (精通) | Python (熟练) | C++ (了解)
游戏引擎：Unity 3D | Unreal Engine 4
数据库：MySQL | Redis | MongoDB
工具/平台：Git | Docker | Jenkins | AWS
其他：帧同步/状态同步 | 性能优化 | 服务器架构设计
```

要点：
- ✅ 分类清晰
- ✅ 标注熟练程度（别全写精通）
- ✅ 只写与岗位相关的

**6. 教育背景**（简洁）

格式：
```
学校名称 | 专业 | 学历 | 时间范围
- 相关课程（可选，应届写）
- 奖项/荣誉（可选，有就写）
```

要点：
- 工作 3 年以上：放最后，简写
- 应届/初级：可放工作经历前面，详细写

---

### 3. 不同经验水平的侧重点

| 部分 | 应届/初级 | 中级 (3-5 年) | 高级 (5 年+) |
|------|-----------|---------------|--------------|
| 个人概述 | 学习能力 + 热情 | 技术深度 + 成果 | 架构能力 + 影响力 |
| 工作经历 | 实习/课程项目 | 核心模块开发 | 架构设计/团队管理 |
| 项目经历 | 参与部分功能 | 独立负责模块 | 主导整体方案 |
| 技能清单 | 基础技能 + 学习意愿 | 专业技能深度 | 技术广度 + 前瞻性 |
| 教育背景 | 详细写（课程/成绩） | 简写 | 一行带过 |

---

### 4. 常见错误（提醒用户避免）

| 错误 | 问题 | 修正 |
|------|------|------|
| 写"负责开发游戏功能" | 太模糊 | "负责战斗系统开发，支持 10v10 实时对战" |
| 只写职责不写成果 | 看不出做得怎么样 | 加数据："性能提升 30%" |
| 技能全写"精通" | 不真实 | 分级：精通/熟练/了解 |
| 项目写太技术 | HR 看不懂 | 先写 1 句项目简介 |
| 简历超过 2 页 | 重点不突出 | 删减无关经历 |
| 有错别字/格式乱 | 态度问题 | 仔细检查 |

### 5. 提供万能模板（可直接套用）

```markdown
# 姓名 | 求职意向 | 城市
电话 | 邮箱 | GitHub/作品集

## 个人概述
[X 年] 游戏 [岗位] 开发经验，精通 [核心技术栈]，主导/参与过 [X] 款上线项目（[品类]）。
擅长 [核心能力 1]、[核心能力 2]，有 [差异化优势/亮点]。

## 工作经历
**公司 A** | 职位 | 时间
- 负责 [核心职责]
- 成果 1（量化数据）
- 成果 2（量化数据）

**公司 B** | 职位 | 时间
- 负责 [核心职责]
- 成果 1（量化数据）

## 项目经历
**项目 1** | 你的角色 | 时间 | 团队规模
项目简介：[1 句话说明是什么]
技术栈：[列出核心技术]
关键贡献：
- [你做了什么 + 难点 + 解决 + 结果]
- [同上]

**项目 2** | ...

## 技能清单
编程语言：[语言 + 熟练度]
游戏引擎：[引擎名]
数据库/工具：[具体名称]
其他：[领域知识]

## 教育背景
学校 | 专业 | 学历 | 时间
[奖项/课程，可选]
```

---

### 6. 自检清单（让用户对照检查）

- [ ] 求职意向明确具体
- [ ] 个人概述 3-5 句话，有亮点
- [ ] 工作经历用动词开头，有量化成果
- [ ] 项目经历有简介、技术栈、你的贡献
- [ ] 技能分类清晰，熟练度真实
- [ ] 没有错别字，格式统一
- [ ] 控制在 1-2 页
- [ ] 导出为 PDF（别发 Word）

---

## 输出要求

1. **文档开头加标记**：`> 🦐 **三毛生成** | 日期时间 | 版本说明`
2. **用表格对比**：不同经验水平的写法差异用表格展示
3. **给具体示例**：每个部分都要有正面示例
4. **标记常见错误**：用 ❌ 和 ✅ 对比
5. **提供可套用模板**：用户可以直接填空使用

---

## 与其他技能协作

- 如果用户说"帮我优化这份简历"并附上内容 → 转 `resume-optimizer` 生成精美 HTML
- 如果用户说"帮我写简历"但没有内容 → 用本技能给结构指导，然后引导用户填写
- 如果用户需要发布到招聘平台 → 可配合 `baoyu-post-to-x` 等发布技能

---

## 注意事项

- 本技能是**指导性质**，不是直接代写
- 鼓励用户自己填写内容，这样更真实
- 如果用户提供了现有简历，先诊断结构问题再给建议
- 游戏行业简历要突出：项目经验、技术栈、上线成果
